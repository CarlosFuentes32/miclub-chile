# Checklist Go / No Go

## Código

- [ ] Rama candidata confirmada.
- [ ] Commit candidato confirmado.
- [ ] Build completo aprobado.
- [ ] Prisma generate aprobado.
- [ ] Prisma validate aprobado.
- [ ] Migraciones revisadas.
- [ ] Tests enterprise aprobados.
- [ ] Playwright staging aprobado.
- [ ] npm audit sin vulnerabilidades.
- [ ] Sin cambios locales.

## Infraestructura

- [ ] Variables requeridas productivas existen.
- [ ] Secretos no expuestos.
- [ ] Backup productivo creado.
- [ ] Restore temporal validado.
- [ ] DNS productivo intacto.
- [ ] SSL válido.
- [ ] Health live/ready/detail OK.
- [ ] Monitoreo activo.
- [ ] Resend validado con QA interno.
- [ ] Logs/auditoría revisados.

## Producto

- [ ] Landing.
- [ ] Cliente.
- [ ] Comercio.
- [ ] Cajero.
- [ ] Administrador.
- [ ] Registro/login.
- [ ] Compra/puntos/sellos/cashback.
- [ ] Recompensa/canje/anulación.
- [ ] Soporte.
- [ ] Billing manual.

## Comercial

- [ ] Plan piloto definido.
- [ ] Máximo 5 comercios.
- [ ] Precio aprobado por dueño.
- [ ] Trial/gracia aprobados.
- [ ] Método de pago manual definido.
- [ ] Soporte y SLA piloto definidos.
- [ ] Contrato/documentos en revisión.
- [ ] Onboarding listo.

Resultado:

- GO: todos los puntos críticos completos.
- GO CON CONDICIONES: solo pendientes no bloqueantes.
- NO GO: cualquier punto crítico incompleto.
