# Proceso de cobro manual

1. Comercio recibe instrucciones de transferencia.
2. Comercio informa referencia/comprobante por canal oficial.
3. Super Admin registra pago como `pending`.
4. Responsable revisa evidencia fuera del sistema.
5. Super Admin aprueba o rechaza.
6. Pago aprobado activa o renueva.
7. Se registra evento financiero.
8. Se envía notificación si Resend está operativo.

Importante: pago registrado no equivale a boleta/factura. Documento tributario se gestiona por proceso administrativo separado.
