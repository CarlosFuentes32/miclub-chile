# Impersonation limitada de soporte

La impersonation de soporte queda deshabilitada en Sprint 6.

Motivo: aún falta definir aprobación formal, banner persistente y bloqueo verificable de acciones sensibles como billing, cambios de rol, exportaciones, backups, cambios de contraseña y operaciones financieras.

El endpoint actual registra la solicitud y responde `enabled: false`.
